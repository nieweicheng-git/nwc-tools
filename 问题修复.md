小工具百宝箱页面UI BUG清单
针对你提供的实际页面截图，严格对照《全页面 UI 设计文档_极致详细版》，逐像素核查 UI 视觉问题，极致细致到每个元素的尺寸、样式、对齐、位置：

---
一、整体页面基础 UI BUG
1. 背景样式完全偏离
  - 问题：未使用设计文档规定的 `linear-gradient(135deg, #F5F7FA 0%, #E4E8F0 100%)` 柔和渐变背景，当前为单调浅灰纯色，无任何分层毛玻璃通透感，无深度感
  - 违反规范：设计文档要求实现「背景层 / 容器层 / 内容层」三层分层视觉，当前完全缺失
  - 正确标准：页面需呈现柔和渐变 + 毛玻璃通透效果，营造高级深度感
2. 页面边距与布局偏离
  - 问题：页面上下内边距≠40px、左右内边距≠24px，顶部导航栏几乎贴到浏览器顶部，无顶部留白，大屏下内容撑度过大，无标准留白
  - 违反规范：设计文档规定 PC 端页面最大宽度 1200px，上下内边距 40px、左右内边距 24px，内容居中
  - 正确标准：大屏下内容居中锁定 1200px，与浏览器边缘保留标准间距，页面透气不拥挤
3. 冗余 UI 元素过多
  - 问题：页面中间显示「早上好 今天需要什么工具？」「当前展示 19 个 共 19 个可用」「1 个自定义」等多余提示文字，无业务意义，破坏页面整洁
  - 违反规范：设计文档要求页面仅保留导航、分类、工具卡片核心内容，无任何无关冗余提示
  - 正确标准：保留正确标准文字、放在一行，同时压缩上下间距，确保上下文流畅
4. 主题切换 UI 完全缺失
  - 问题：无设计文档规定的明暗主题切换开关，无主题切换 UI 入口
  - 违反规范：设计文档要求导航栏右侧添加主题开关，支持 0.4s 平滑过渡
  - 正确标准：导航栏右侧添加主题切换开关，样式统一，支持明暗主题切换

---
二、顶部导航栏 UI BUG
1. 导航栏尺寸与样式偏离
  - 问题：导航栏高度≠56px，当前过高；无设计文档规定的毛玻璃半透明背景，当前为纯白色；圆角≠28px，当前仅 16px，过小；无标准轻量阴影 `0 2px 8px rgba(0,0,0,0.06)`
  - 违反规范：设计文档规定导航栏固定高度 56px，毛玻璃背景、28px 大圆角、标准阴影
  - 正确标准：导航栏尺寸、背景、圆角、阴影严格符合规范，实现毛玻璃悬浮效果
2. 搜索框 UI 完全偏离
  - 问题：搜索框宽度≠400px，当前过宽，撑满了导航栏大部分空间；高度≠40px，当前过高；圆角≠20px，过小；占位符「搜索工具...」未居中，靠左对齐；无毛玻璃半透明背景
  - 违反规范：设计文档规定搜索框宽度 400px、高度 40px、圆角 20px，占位符文字居中，毛玻璃背景
  - 正确标准：搜索框尺寸、样式统一，占位符居中，与导航栏风格统一
3. 元素对齐与布局混乱
  - 问题：Logo、搜索框、添加按钮未按设计居中对齐，搜索框过宽导致元素间距混乱；导航栏右侧缺失主题开关
  - 违反规范：设计文档要求导航栏元素水平居中对齐，Logo 在左、搜索框居中、主题开关在右
  - 正确标准：导航栏元素布局规范，无重复按钮，主题开关补全
4. 吸顶效果 UI 缺失
  - 问题：无设计文档规定的滚动 100px 自动吸顶效果，无吸顶过渡动画
  - 违反规范：设计文档要求滚动时导航栏自动吸顶，伴随 0.3s 平滑过渡
  - 正确标准：滚动超过 100px 时，导航栏固定在顶部，过渡平滑

---
三、分类栏 UI BUG（核心问题）
1. 分类栏尺寸与样式偏离
  - 问题：分类栏高度≠48px，当前过高；无设计文档规定的毛玻璃半透明背景，当前为纯白色；圆角≠24px，当前仅 16px，过小；无标准轻量阴影
  - 违反规范：设计文档规定分类栏固定高度 48px，毛玻璃背景、24px 圆角、轻量阴影
  - 正确标准：分类栏样式与导航栏风格统一，实现毛玻璃悬浮效果
2. 分类标签尺寸与间距混乱
  - 问题：单个标签高度≠36px，当前过高；内边距≠`上下8px、左右16px`，当前左右内边距过大，导致标签过宽；标签间距≠8px，当前间距 12px，过宽；标签圆角≠18px，当前仅 12px，过小；且因标签尺寸、内边距设置不合理，部分标签文字存在展示不全、挤压问题，未按设计要求自适应文字宽度，长分类文字被挤压变形
  - 违反规范：设计文档规定标签最小宽度 80px、最大宽度 140px，高度 36px，内边距`8px 16px`，间距 8px，圆角 18px，自适应文字宽度
  - 正确标准：标签尺寸、内边距、间距统一，文字长度不同时标签宽度自适应，无拥挤 / 空白问题
3. 分类选中态 UI 偏离
  - 问题：选中标签未使用设计文档规定的渐变背景，当前为纯色红色背景；文字样式未按设计渐变高亮，仅为白色文字，无渐变效果
  - 违反规范：设计文档规定选中标签使用渐变背景 + 白色加粗文字，与普通标签区分
  - 正确标准：选中标签使用渐变高亮背景，视觉突出，与普通标签区分明显
4. 分类栏重复与冗余 UI
  - 问题：分类栏多了「自研」「第三方」「全部」三个多余分类，设计文档规定分类为功能分类（推荐 / 开发 / 设计 / 办公 / 系统 / 娱乐），来源是卡片标签，不是分类；同时导航栏和分类栏都有「添加」按钮，重复 UI
  - 违反规范：设计文档规定分类栏仅保留功能分类，来源标签放在卡片右上角，无重复添加按钮
  - 修复建议：删除分类栏的「添加工具」按钮，保留导航栏右侧的「添加」按钮，避免重复 UI；同时删除多余的来源分类，仅保留功能分类
  - 正确标准：分类栏仅保留功能分类，删除多余的来源分类，删除重复的添加按钮
5. 悬停态 UI 缺失
  - 问题：分类标签无设计文档规定的悬停浅灰背景、0.25s 过渡动画，无任何悬停反馈
  - 违反规范：设计文档规定悬停标签有丝滑反馈，与普通标签区分
  - 正确标准：悬停标签有浅灰背景，过渡平滑，交互反馈明确
6. 分类切换动画效果缺失
  - 问题：分类标签切换时，无设计文档规定的动画效果，切换后工具卡片直接生硬刷新、跳变，无任何平滑过渡，视觉体验卡顿、不流畅
  - 违反规范：设计文档要求分类切换时，工具卡片需伴随0.3s淡入淡出动画，分类标签切换时也需有0.25s平滑过渡，提升交互流畅度
  - 正确标准：添加分类切换动画，标签切换时有丝滑过渡，工具卡片随分类切换实现0.3s淡入淡出效果，无生硬跳变，交互体验流畅

---
四、工具卡片 UI BUG（核心体验问题）
1. 卡片尺寸与样式偏离
  - 问题：卡片内边距≠24px，当前过小；圆角≠20px，当前仅 16px，过小；初始阴影≠`0 2px 8px rgba(0,0,0,0.06)`，当前几乎无阴影，卡片扁平无深度
  - 违反规范：设计文档规定 PC 端卡片固定宽度 240px，内边距 24px，圆角 20px，初始轻量阴影
  - 正确标准：卡片尺寸、圆角、阴影严格符合规范，有悬浮深度感
2. 卡片结构完全混乱（重点修复项）
  - 问题（重点）：所有按钮（打开 / 删除）都放在卡片外部，单独占一行，卡片本身仅包含图标 / 文字，严重破坏了卡片的完整性，导致页面出现「卡片一行、按钮一行」的割裂布局，视觉杂乱，严重影响用户体验，属于核心UI布局问题，需优先修复。
  - 违反规范（重点强调）：设计文档明确规定，所有卡片元素（图标 / 文字 / 按钮 / 删除按钮）必须全部包含在卡片内部，卡片作为一个完整的独立模块，不允许拆分元素至外部，确保布局统一、视觉连贯。
  - 修复要求及正确标准（重点明确）：必须将所有按钮（打开/删除）迁移至卡片内部，与图标、文字等元素整合，确保卡片是一个完整的独立模块，无任何外部拆分元素；按钮需按设计规范布局在卡片内部底部，与其他元素协调统一，彻底解决布局割裂问题。
3. 卡片全域点击 UI 缺失
  - 问题：卡片主体区域无点击效果，仅外部的「打开」按钮可点击，用户点击卡片主体无任何反馈，完全不符合设计
  - 违反规范：设计文档规定整个卡片为点击区域，任意位置点击均可触发跳转 / 下载
  - 正确标准：卡片全域可点击，点击任意位置均可触发操作，用户无需精准点按钮
4. 来源标签 UI 偏离
  - 问题：来源标签（自研 / 第三方）圆角≠8px，当前仅 4px，过小；字体≠11px，当前过大；内边距≠`3px 8px`，当前过大；背景样式未按设计区分（自研渐变、第三方浅灰），当前仅为纯色小标签
  - 违反规范：设计文档规定来源标签固定在卡片右上角，圆角 8px，字体 11px，内边距`3px 8px`，样式区分来源
  - 正确标准：来源标签样式统一，位置固定，与卡片其他元素区分
5. 删除按钮 UI 完全偏离
  - 问题：删除按钮默认显示，放在卡片外部，每个卡片下面都有一个垃圾桶图标，页面上一堆冗余图标，极度混乱；位置不在卡片右下角，默认显示，易误触
  - 违反规范：设计文档规定删除按钮仅在卡片悬停时显示，固定在卡片右下角，平时隐藏
  - 正确标准：删除按钮默认隐藏，悬停卡片时才出现，位置合理，不破坏页面整洁
6. 卡片间距与动效缺失
  - 问题：卡片间距≠28px，当前仅 20px，过挤；无设计文档规定的悬停上浮 4px、阴影加深的动效，无任何悬停反馈
  - 违反规范：设计文档规定卡片间距 28px，悬停时上浮 4px，伴随 0.25s 平滑过渡
  - 正确标准：卡片间距合理，悬停时有上浮反馈，过渡平滑
7. 按钮 UI 完全偏离
  - 问题：「使用 / 下载」按钮放在卡片外部，样式为原生按钮，未按设计放在卡片内部底部居中，无渐变文字样式
  - 违反规范：设计文档规定按钮在卡片内部底部居中，文字为渐变样式，引导用户
  - 正确标准：按钮在卡片内部，样式统一，渐变文字，视觉突出
8. 添加工具页面分类与工具来源未对齐
  - 问题：添加工具页面中，「分类」选项与「工具来源」选项未实现左右对齐，两者在水平方向存在偏移（可能是标签宽度不一致、内边距不同或布局方式错误），视觉上杂乱，不符合UI统一规范
  - 违反规范：设计文档要求添加工具页面所有表单选项（含分类、工具来源）需水平对齐，标签、输入框/下拉框的对齐方式统一，保持页面整洁有序
  - 正确标准：「分类」与「工具来源」选项标签宽度一致、内边距统一，输入框/下拉框水平对齐，无水平偏移，视觉上整齐规范
9. 自己添加的工具页面显示异常（重点修复项）
  - 问题（重点）：用户自行添加的工具，在页面展示时存在明显异常，具体表现为：添加的工具卡片样式与原有工具卡片不统一、布局错乱，部分添加的工具无图标/文字缺失，工具信息（名称、描述、来源）展示不完整、错位，甚至出现添加后不显示、显示重复或显示位置异常的情况，严重影响使用体验，属于核心功能相关UI异常，需优先修复。
  - 违反规范（重点强调）：设计文档明确规定，用户添加的工具（无论URL类型还是文件类型），展示样式需与原有工具卡片完全统一，信息完整、布局规范，无任何显示错位、缺失或重复问题，确保页面视觉一致性。
  - 修复要求及正确标准（重点明确）：必须统一用户添加工具与原有工具的卡片样式，确保添加后工具卡片尺寸、圆角、阴影、内边距与原有卡片一致；工具信息（名称、描述、来源、按钮）完整显示、无错位，添加后能正常展示在对应分类下，无缺失、重复、位置异常问题，与页面整体UI风格统一。
10. 分页功能缺失（需新增）
  - 问题：页面未实现分页功能，所有工具卡片全部一次性展示，未按要求限制每页最多显示10个工具，当工具数量增多时，页面过长、滚动体验差，且加载卡顿，不符合用户浏览习惯
  - 违反规范：设计文档要求新增分页功能，严格控制每页工具卡片数量≤10个，实现分页切换，提升页面加载速度与浏览体验
  - 修复要求及正确标准：新增分页功能，默认每页显示10个工具卡片，超出部分分页展示；分页控件需与页面UI风格统一（毛玻璃、圆角、统一字体），包含页码切换、上一页/下一页按钮，切换时伴随0.3s丝滑过渡，无生硬跳变；支持页码快速跳转，分页状态清晰可见，确保工具数量增多时页面浏览流畅、加载高效

五、核心 UI 验收未达标项
1. 视觉样式：无任何设计文档规定的渐变、毛玻璃、分层阴影、大圆角，UI 极度扁平简陋
2. 页面结构：卡片结构割裂（重点修复项），按钮在外部，破坏了卡片的完整性，布局混乱，需优先整改
3. 交互 UI：无任何悬停、过渡、吸顶 UI 动效，全部生硬无反馈
4. 冗余 UI：大量重复按钮、多余提示文字，页面整洁度不达标
5. 元素对齐：搜索框占位符、标签布局、元素间距全部混乱，未按设计规范
6. 状态区分：选中态、添加态、普通态标签样式混淆，用户无法区分
7. 表单对齐：添加工具页面「分类」与「工具来源」未左右对齐，视觉杂乱，不符合UI统一规范
8. 切换动画：分类切换时无任何动画效果，工具卡片生硬刷新，不符合交互流畅性规范
9. 添加工具显示：自己添加的工具页面显示异常（重点修复项），卡片样式不统一、信息缺失/错位，需优先整改
10. 分页功能：未新增分页功能，未实现每页最多显示10个工具，工具过多时页面过长、加载卡顿，需按要求新增分页
