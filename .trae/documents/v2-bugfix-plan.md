## 二、需要修复的项（共4项）

### 修复项1：卡片全域点击（BUG 4.3 / UI-CARD-003）— P0

**问题**：仅按钮可点击，卡片主体（图标、名称、描述区域）不可点击

**修复方案**：
- 在 `render_tool_card()` 的卡片 HTML 中，为有URL的在线/下载工具添加全卡片透明链接覆盖层
- 对于在线工具（有URL且不以#开头）：添加 `<a class="card-click-overlay" href="{url}" target="_blank">`
- 对于文件下载工具：不添加覆盖层（下载需要点击下载按钮）
- 对于自研工具（URL以#开头）：不添加覆盖层（无外部链接）
- CSS：`.card-click-overlay { position:absolute; inset:0; z-index:1; border-radius:20px; }`
- 来源标签和删除按钮需 `z-index:2` 确保可点击

**影响范围**：`render_tool_card()` 函数 + CSS 卡片样式

### 修复项2：导航栏高度验证与调整（BUG 6.1 / UI-NAV-001）— P1

**问题**：BUG清单要求导航栏高度56px

**修复方案**：
- 当前代码 `padding:8px 20px` + 内部元素 `height:40px`，总高度约56px
- 启动服务后测量实际高度，如不足则调整 padding 为 `padding:10px 20px` 或添加 `min-height:56px`

**影响范围**：CSS 导航栏样式

### 修复项3：弹窗毛玻璃遮罩（BUG 7.2 / UI-MODAL-001）— P1

**问题**：弹窗无毛玻璃遮罩

**修复方案**：
- 添加弹窗遮罩毛玻璃 CSS：`[data-testid="stModalBackdrop"] { backdrop-filter:blur(8px); background:rgba(0,0,0,0.3)!important; }`

**影响范围**：CSS 弹窗样式

### 修复项4：添加工具页面所有字段必填+详细提示语（BUG 7.3 / 用户新需求）— P1

**问题**：添加工具弹窗中部分字段非必填，placeholder提示语不够详细

**修复方案**：

**当前代码问题分析**：
1. 「工具名称」有 `max_chars=20` 和 placeholder，但无必填校验提示
2. 「工具描述」有 `max_chars=50` 和 placeholder，但无必填校验提示
3. 「工具链接」有 placeholder，但无必填校验提示
4. 「标签」无必填要求，无提示
5. 文件上传模式下，链接和来源/使用类型字段缺失
6. 提交按钮点击后才校验，校验提示不够友好

**具体修改**：
1. 所有 `st.text_input` 添加详细 placeholder：
   - 工具名称：`"请输入工具名称（必填，2-20个字符）"`
   - 工具描述：`"请输入工具描述（必填，5-50个字符，一句话说明用途）"`
   - 工具链接：`"请输入工具链接（必填，以http://或https://开头）"`
   - 标签：`"请输入标签（必填，逗号分隔，最多3个，每个最多4字）"`
2. 提交时增加前端校验，空字段显示友好提示：
   - 名称：`"⚠️ 请输入工具名称"`
   - 描述：`"⚠️ 请输入工具描述"`
   - 链接：`"⚠️ 请输入工具链接"`
   - 标签：`"⚠️ 请输入至少一个标签"`
3. 文件上传模式下，文件为必填项：`"⚠️ 请选择要上传的文件"`
4. 所有 label 添加 `*` 标记表示必填

**影响范围**：`render_add_modal()` → `add_dialog()` 函数
