# 小工具百宝箱页面BUG清单（对照《全页面UI设计文档_极致详细版》核查）
> 说明：本清单严格按照设计文档规范逐条核查，覆盖像素、文字、交互、样式、响应式全维度，所有BUG均标注「问题描述/违反规范/正确标准」，可直接交付开发修复。

---

## 一、整体页面基础BUG
- **BUG 1：页面背景层级缺失**
  - 问题描述：未使用文档规定的渐变背景，无分层毛玻璃通透感，当前为单调纯色，无大气舒展感
  - 违反规范：文档规定背景使用 `linear-gradient(135deg, #F5F7FA 0%, #E4E8F0 100%)`（浅色）/ `linear-gradient(135deg, #0F0F12 0%, #1A1A22 100%)`（深色），且需实现背景层/容器层/内容层三层分层
  - 正确标准：页面需呈现柔和渐变+毛玻璃通透效果，营造深度感

- **BUG 2：页面布局未居中，边距不标准**
  - 问题描述：页面未锁定最大宽度，内容未居中显示，上下/左右内边距不符合规范
  - 违反规范：文档规定PC端页面最大宽度为1200px，上下内边距40px、左右内边距24px，需居中显示
  - 正确标准：大屏下内容居中锁定1200px，与浏览器边缘保留标准间距，无贴边/溢出问题

- **BUG 3：主题切换无平滑过渡**
  - 问题描述：明暗主题切换时出现颜色跳变、闪烁，无过渡效果
  - 违反规范：文档规定主题切换需添加0.4s全元素平滑过渡
  - 正确标准：切换主题时，所有背景、文字、组件颜色同步渐变，无生硬闪烁

## 二、顶部导航栏BUG
- **BUG 1：导航栏尺寸与样式不规范**
  - 问题描述：导航栏高度≠56px，无标准毛玻璃背景，圆角≠28px，无轻量阴影
  - 违反规范：文档规定导航栏固定高度56px，背景为 `rgba(255,255,255,0.7)`（浅色）/ `rgba(30,30,36,0.7)`（深色），圆角28px，阴影为 `0 2px 8px rgba(0,0,0,0.06)`
  - 正确标准：导航栏尺寸、背景、圆角、阴影需严格符合规范，实现毛玻璃悬浮效果

- **BUG 2：导航栏元素对齐混乱**
  - 问题描述：Logo、搜索框、主题开关未水平居中对齐，搜索框尺寸不标准
  - 违反规范：文档规定搜索框宽度400px、高度40px、圆角20px，占位符文字需居中显示，所有导航元素需在56px高度内垂直居中
  - 正确标准：所有导航元素垂直居中对齐，搜索框尺寸、样式统一，占位符文字居中

- **BUG 3：吸顶效果失效**
  - 问题描述：页面滚动超过100px时，导航栏未触发吸顶效果，或吸顶无过渡动画
  - 违反规范：文档规定滚动超过100px时，导航栏自动吸顶，伴随0.3s平滑过渡，毛玻璃背景需同步生效
  - 正确标准：滚动超过100px时，导航栏固定在顶部，过渡平滑，背景不失效

- **BUG 4：冗余按钮未删除**
  - 问题描述：页面仍保留「重置默认工具」相关逻辑，未按需求删除
  - 违反规范：文档明确要求删除「重置为默认工具」功能，无相关按钮/逻辑
  - 正确标准：页面无任何重置默认工具的入口与逻辑

## 三、分类栏核心BUG（重点问题）
- **BUG 1：分类栏基础样式不规范**
  - 问题描述：分类栏高度≠48px，无毛玻璃背景，圆角≠24px，无标准阴影
  - 违反规范：文档规定分类栏固定高度48px，毛玻璃背景，圆角24px，轻量阴影
  - 正确标准：分类栏样式与导航栏风格统一，实现悬浮毛玻璃效果

- **BUG 2：分类标签尺寸与间距混乱**
  - 问题描述：单个标签高度≠36px，内边距≠`上下8px、左右16px`，标签间距≠8px，拥挤/过宽，未按文字自适应宽度
  - 违反规范：文档规定标签最小宽度80px、最大宽度140px，高度36px，内边距`8px 16px`，间距8px，需根据文字长度自适应宽度
  - 正确标准：标签尺寸、内边距、间距统一，文字长度不同时标签宽度自适应，无拥挤/空白问题

- **BUG 3：文字未完全居中，出现截断**
  - 问题描述：分类标签文字未水平+垂直完全居中，部分标签文字被截断、出现省略号
  - 违反规范：文档规定所有标签文字需强制完全居中，永不截断，需100%完整展示
  - 正确标准：标签文字居中对齐，所有文字完整显示，无省略号、截断问题

- **BUG 4：选中/悬停态样式缺失**
  - 问题描述：选中标签无渐变高亮背景，悬停无反馈效果，文字样式未变化
  - 违反规范：文档规定选中标签使用渐变背景+白色加粗文字，悬停时添加浅灰背景+0.25s过渡动画
  - 正确标准：选中标签高亮突出，悬停标签有丝滑反馈，与普通标签区分明显

- **BUG 5：「添加工具」标签样式不突出**
  - 问题描述：「添加工具」标签无特殊高亮样式，前置➕图标缺失，文字未居中
  - 违反规范：文档规定「添加工具」标签需使用特殊渐变样式，前置➕图标，文字居中显示，与其他标签区分
  - 正确标准：「添加工具」标签视觉突出，引导用户操作

- **BUG 6：冗余提示未删除**
  - 问题描述：页面仍显示「已隐藏X个工具」相关提示文字，未按需求删除
  - 违反规范：文档明确要求删除隐藏数量提示，页面无相关内容
  - 正确标准：页面无任何工具隐藏数量提示

## 四、工具卡片核心BUG（核心体验问题）
- **BUG 1：卡片尺寸与样式不规范**
  - 问题描述：卡片宽度≠240px，内边距≠24px，圆角≠20px，初始阴影与悬停阴影不标准
  - 违反规范：文档规定PC端卡片固定宽度240px，内边距24px，圆角20px；初始阴影为 `0 2px 8px rgba(0,0,0,0.06)`，悬停阴影为 `0 8px 24px rgba(99,102,241,0.15)`
  - 正确标准：卡片尺寸、圆角、阴影严格符合规范，悬停时有上浮效果

- **BUG 2：卡片未实现全域点击**
  - 问题描述：仅「使用/下载」按钮可点击，卡片主体区域无点击跳转效果，不符合文档设计
  - 违反规范：文档规定整个卡片为点击区域，任意位置点击均可触发工具跳转/下载
  - 正确标准：卡片全域可点击，点击任意位置均可触发对应操作，跳转无延迟

- **BUG 3：卡片内文字对齐混乱，出现截断**
  - 问题描述：工具名称、描述、标签、按钮文字未水平居中，部分文字被截断、换行，未单行完整展示
  - 违反规范：文档规定除删除按钮外，所有卡片内文字需水平居中，永不截断，保持单行完整展示
  - 正确标准：卡片内所有文字居中对齐，无截断、换行、省略号问题

- **BUG 4：来源标签样式不规范**
  - 问题描述：「自研/第三方」标签位置错误，圆角≠8px，字体≠11px，内边距≠`3px 8px`，文字未居中
  - 违反规范：文档规定来源标签固定在卡片右上角，圆角8px，字体11px，内边距`3px 8px`，文字居中显示
  - 正确标准：来源标签样式统一，位置固定，与卡片其他元素区分

- **BUG 5：删除按钮交互逻辑错误**
  - 问题描述：删除按钮默认显示，非悬停触发，位置不在卡片右下角，易误触
  - 违反规范：文档规定删除按钮仅在卡片悬停时显示，固定在卡片右下角，避免误触
  - 正确标准：删除按钮默认隐藏，悬停卡片时出现，位置合理，不易误触

- **BUG 6：悬停动效缺失**
  - 问题描述：卡片悬停无上浮、缩放效果，无过渡动画，交互反馈生硬
  - 违反规范：文档规定卡片悬停时需上浮4px，伴随0.25s平滑过渡，点击时缩放0.98
  - 正确标准：卡片悬停有明显上浮反馈，过渡平滑，点击有按压效果

- **BUG 2：弹窗样式与动效缺失**

  - 正确标准：按钮布局合理，样式统一，提交按钮视觉突出，引导用户操作

## 六、交互细节BUG
- **BUG 1：分类切换无平滑过渡**
  - 问题描述：切换分类时，工具卡片直接刷新，无淡入淡出过渡动画，生硬跳变
  - 违反规范：文档规定分类切换时，卡片需伴随0.3s淡入淡出过渡
  - 正确标准：切换分类时，卡片平滑淡入淡出，无生硬刷新

- **BUG 2：搜索过滤无防抖与过渡**
  - 问题描述：搜索输入时，实时触发过滤，无300ms防抖，卡片无过渡效果，卡顿跳变
  - 违反规范：文档规定搜索需添加300ms防抖，过滤结果需伴随淡入淡出过渡
  - 正确标准：搜索输入时，防抖触发过滤，卡片平滑更新，无卡顿

- **BUG 3：小屏分类栏无法横向滚动**
  - 问题描述：小屏幕下，分类标签超出容器，无法横向滚动，未隐藏滚动条
  - 违反规范：文档规定小屏下分类栏支持横向滚动，滚动条隐藏
  - 正确标准：小屏下分类标签可横向滑动，无滚动条溢出问题

- **BUG 4：增删工具后状态重置**
  - 问题描述：添加/删除工具后，当前选中的分类、搜索状态被重置，用户体验断裂
  - 违反规范：文档规定增删工具后，需保留用户当前的分类选择、搜索状态
  - 正确标准：增删工具后，页面状态不重置，用户无需重复操作

## 九、Streamlit托管页面（nietools.streamlit.app）专属BUG
### 9.1 框架与技术架构BUG
- **BUG 1：技术框架不符合设计要求**
  - 问题描述：采用Streamlit框架开发，非原生Web前端实现，完全无法适配设计文档的自定义UI、交互动效、布局规范
  - 违反规范：文档明确为**纯Web端（PC浏览器）原生前端**设计，不支持Streamlit低代码框架约束
  - 正确标准：改用原生HTML/CSS/JS/前端框架实现，完全遵循设计文档规范

- **BUG 2：部署方式错误**
  - 问题描述：使用Streamlit官方免费托管，非Cloudflare Pages部署，无法实现24小时免费稳定运行、电脑关机访问
  - 违反规范：要求采用**Cloudflare Pages + Cloudflare R2 + Cloudflare Workers + Supabase**全家桶部署
  - 正确标准：迁移至Cloudflare全家桶部署，实现永久免费、24小时在线

### 9.2 页面结构与视觉BUG
- **BUG 1：无标准页面结构**
  - 问题描述：无设计文档规定的**顶部导航栏、分类栏、工具卡片网格布局**，全程Streamlit原生组件堆砌，结构完全混乱
  - 违反规范：文档第2-5部分明确规定完整页面结构
  - 正确标准：严格按照导航栏→分类栏→工具卡片流的结构布局

- **BUG 2：无自定义视觉样式**
  - 问题描述：全程使用Streamlit原生底色、原生组件，无渐变、毛玻璃、分层阴影、大圆角等设计元素，UI极度简陋
  - 违反规范：文档规定全套高级视觉样式规范
  - 正确标准：实现柔和渐变、毛玻璃、悬浮阴影、统一大圆角的高级视觉效果

- **BUG 3：无关冗余提示文字**
  - 问题描述：页面顶部显示`Status embed installed...`无关提示文字，无业务意义，影响页面整洁度
  - 违反规范：页面仅保留工具核心内容，无任何无关冗余文字
  - 正确标准：删除所有无关状态提示，页面仅展示工具百宝箱核心内容

### 9.3 核心功能缺失BUG
- **BUG 1：无工具分类与筛选功能**
  - 问题描述：无分类栏、无分类切换、无搜索过滤，无法按自研/第三方/分类筛选工具
  - 违反规范：文档第3、4、7部分明确分类、搜索功能规范
  - 正确标准：实现分类切换、搜索过滤、平滑过渡筛选效果

- **BUG 2：无标准工具卡片功能**
  - 问题描述：无设计文档规定的工具卡片，无卡片全域点击、悬停动效、来源标签、删除按钮等功能
  - 违反规范：文档第5部分工具卡片全规范
  - 正确标准：实现标准工具卡片，支持全域点击、悬停反馈、文字居中完整展示

- **BUG 3：无文件上传/下载核心功能**
  - 问题描述：不支持本地py脚本/文件上传、在线下载，仅支持URL工具，未满足核心需求
  - 违反规范：要求支持**文件上传+Cloudflare R2存储+在线下载**
  - 正确标准：对接Cloudflare R2，实现文件上传、存储、下载全流程

- **BUG 4：无工具增删管理功能**
  - 问题描述：无「添加新工具」弹窗、无工具删除功能，无法自定义管理工具列表
  - 违反规范：文档第6部分明确添加工具弹窗、增删功能规范
  - 正确标准：实现添加工具弹窗、工具删除、数据持久化存储

- **BUG 5：无数据持久化存储**
  - 问题描述：未对接Supabase数据库，工具数据无持久化，刷新页面数据丢失，换设备无法同步
  - 违反规范：要求使用Supabase实现数据永久存储、多端同步
  - 正确标准：对接Supabase，实现工具数据增删改查、永久不丢失

### 9.4 交互与体验BUG
- **BUG 1：无任何自定义交互动效**
  - 问题描述：无卡片悬停、无弹窗动效、无吸顶、无平滑过渡，全程Streamlit原生生硬交互
  - 违反规范：文档第7部分全套交互细节规范
  - 正确标准：实现所有丝滑交互动效，符合设计文档时长与反馈规则

- **BUG 2：响应式布局完全失效**
  - 问题描述：采用Streamlit原生流式布局，无按设计文档的列数适配（大屏5列/中屏4/3列/小屏2列）
  - 违反规范：文档第8部分响应式布局全规范
  - 正确标准：严格按屏幕宽度实现卡片列数自适应，分类栏小屏滑动

## 十、补充验收未达标项（Streamlit页面）
1.  技术架构：采用非指定框架与部署方式，无法满足永久在线、文件存储、数据持久化需求

3.  核心功能：缺失文件上传、工具增删、分类筛选、数据持久化等核心能力
